# Documentación - L&M PC Computadoras

## Cómo Leer Esta Documentación

### Para Usuarios Finales

1. Lee DOCUMENTACION.md para obtener una visión general
2. Sigue INSTALACION.md para configurar la aplicación
3. Usa GUIA_USUARIO.md como manual de referencia
4. Consulta GUIA_USUARIO.md en casos de preguntas frecuentes

### Para Administradores de Sistema

1. Lee DOCUMENTACION.md para entender la aplicación
2. Sigue INSTALACION.md detalladamente
3. Consulta GUIA_USUARIO.md para la administración de citas
4. Revisa la sección de Solución de Problemas en INSTALACION.md

### Para Desarrolladores

1. Lee DOCUMENTACION.md para contexto
2. Revisa DESARROLLO.md para arquitectura y estándares
3. Consulta API.md para especificaciones de endpoints
4. Usa INSTALACION.md para ambiente local
5. Sigue GUIA_USUARIO.md para flujos de usuario

## Estructura de la Documentación

Todos los documentos siguen esta estructura:

1. Índice o Tabla de Contenidos
2. Secciones principales con títulos claros
3. Subsecciones con información detallada
4. Ejemplos de código cuando es relevante
5. Tablas para información estructurada
6. Consejos y notas importantes

## Convenciones Utilizadas

Títulos: Encabezados jerárquicos (# para principal, ## para secciones)
Código: Bloques de código con lenguaje especificado
Énfasis: Texto en negrita para términos importantes
Listas: Viñetas o números según corresponda
